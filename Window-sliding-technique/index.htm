                   Naive Approach of Time Complexity O(n*k) :

              Checking all possible consecutive blocks of k elements :

                          Algorithm :

maximum_sum(arr[], k, n)

  max_sum = MIN_VALUE

 

  for i = 0 to i <= n-k

    cur_sum = 0

    for j = i to j <= k

       cur_sum += arr[j]

    max_sum = max(max_sum, cur_sum)

 

  return max_sum

               

                             where, n is the number of array elements. 

                   Efficient Approach of Time Complexity O(n) :

                Using Window sliding technique :

                Algorithm :

maxSum(arr[], n, k):

  max_sum = 0

  for i = 0 to i < k   

    max_sum += arr[i]

 

  /* Compute remaining windows sum by

  removing first element of previous

  window and adding last element of

  current window. */

 

  cur_sum = max_sum

  for i = k to i < k

    cur_sum += arr[i] - arr[i - k]

    max_sum = max(cur_sum, max_sum)

  

  return max_sum

                  

                             where, n is the number of array elements.
